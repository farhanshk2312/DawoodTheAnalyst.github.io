Secure Data Modernization Pipeline on Azure

High Level Architecture:

1. On-prem SQL Server (source) ‚Üí Azure Data Factory triggers and orchestrates data movement and Databricks notebooks (ETL).

2. Databricks reads from SQL Server (JDBC) and writes Delta tables into ADLS Gen2 using a Bronze‚ÜíSilver‚ÜíGold lakehouse pattern.

3. Azure Synapse (serverless SQL pool and/or dedicated SQL pool) exposes curated datasets as external tables/views for analytics.

4. Power BI connects to Synapse (DirectQuery or Import) for dashboards.

5. Secrets (JDBC credentials, storage keys, service principal secrets) live in Azure Key Vault; Azure Active Directory manages user/service identities and RBAC.

6. ADF handles scheduling, monitoring, and branch-able pipeline orchestration.

7. Observability via Log Analytics + Azure Monitor. Governance via Key Vault, resource tagging.

Tools:

1. Resource Group(s): rg--env (per env: dev/test/prod)

2. VNet with subnets for Databricks / Synapse.

3. Azure SQL Server (or on-prem SQL accessible via VPN/ExpressRoute/Private Link)

4. Storage Account (ADLS Gen2) with containers: /bronze, /silver, /gold.

5. Azure Databricks workspace (prefer VNet injection / Private Link)

6. Azure Synapse workspace (serverless SQL pool; optionally dedicated SQL pool)

7. Azure Data Factory (ADF) instance

8. Azure Key Vault(s) (separate per env)

9. Managed Identities / Service Principals (for Databricks, Synapse, ADF, Power BI)

10. Log Analytics workspace + Azure Monitor + Alerts
