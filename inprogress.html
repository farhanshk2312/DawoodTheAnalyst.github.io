Secure Data Modernization Pipeline on Azure

High Level Architecture:

Source SQL Server → Azure Databricks (ETL) reads via JDBC (private network / VPN / Private Endpoint) → writes canonical data as Delta layers (Bronze / Silver / Gold) into Azure Data Lake Storage Gen2 (ADLS Gen2) → Azure Synapse (serverless SQL pool for logical/interactive querying, and/or dedicated SQL pool for high-performance DW) exposes relational views / external tables → Power BI connects (DirectQuery or Import) for reporting and dashboards. Secrets live in Azure Key Vault and Databricks reads them via an Azure Key Vault–backed secret scope; identities and user auth are handled by Azure Active Directory (Azure AD).

Tools:

1. Resource Group, Virtual Network (VNet) with subnets (for private connectivity).

2. SQL Server (on-prem/VM/Azure SQL) — reachable over private network or via secure endpoint.

3. Storage account (ADLS Gen2) with hierarchical namespace enabled (containers for bronze/silver/gold).

4. Azure Key Vault(s) for secrets (separate per env: dev/prod).

5. Azure Databricks workspace (VNet injection / Private Link recommended).

6. Azure Synapse workspace (serverless SQL pool; optional dedicated SQL pool).

7. Power BI tenant / app registration (service principal if automating embedding or refresh).

8. AObservability: Log Analytics workspace / Azure Monitor / diagnostic settings.

Showcases:

1. Delta Lake on ADLS Gen2 gives ACID, time travel, and efficient upserts (ideal for CDC/slowly changing tables).

2. Synapse serverless provides a T-SQL logical layer over files (no copy), while dedicated pools give predictable performance for large analytic workloads.

3. Key Vault + Azure AD minimizes secret sprawl and enables RBAC & rotation best practices.
